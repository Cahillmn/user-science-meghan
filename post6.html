Make User Science Journal Accessible

Revising my post 2 blog for accessibility

Posted by Meghan Cahill on May 16, 2020

This week I performed an analysis of my second blog post “Usable Design Analysis", .Using Wave and Lighthouse, I identified issues with accessibility that users would experience with my webpage. Based on these findings, I made improvements to the website to address these issues.

Findings

Lighthouse

In Lighthouse, the page initially had a score of 81 for accessibility.

Three errors were identified.

  1. The images did not contain alt attributes (high priority)
  2. Links did not have a discernible name (low priority)
  3. Lists contain other elements besides list, script and template elenents (low priority)

Wave

Wave had many of the same issues but also found contrast errors in my header. It also noted the underlined text as an alert.

Issues addressed

Use of underlining

This was a problem because underlining often means there’s a hyperlink. I had used it to emphasize text which was not good design. To fix it I removed the underline element from my code.

Missing Alternative Text

None of my images had alternative text. It is important to include for those who have visual impairments and rely on screen readers. The screen reader will read the descriptive text in the alternative text. This makes it clear to the user what the images are. To fix it I added alt text to all of my images.

Empty Links

The page included three placeholder links to Twitter, Facebook, and Github. None of them included the URL to the applicable site. In addition to the URL being missing, there was no text describing the purpose of the link. This would also make it difficult for a screen reader to interpret the purpose of the link. To fix it I added the URL’s and descriptive text.

Contrast Errors

The final error identified by Wave but not Lighthouse was a lack of contrast between the background and foreground of my header. I had used a medium blue colored banner and changed it to a darker one. I still got the error when I re-evaluated the page in Wave. It turns out that Wave was thinking that both the foreground and background were white even though the background was dark blue. This turned out to be a false error that I chose to ignore.

Outcome

After making all of these fixes, I re-evaluated my page in Wave and Lighthouse and saw improvement. To move to the revised site, click here.

Lighthouse

My score increased from 81 to 96. This was an 18.5% improvement.

Wave

All previous errors except the false contrast errors were resolved.

Challenges

My biggest challenge was trying to get the contrast right on my menu. I spent hours playing with css and html to try and change the contrast with no success. Once I confirmed with my teacher that this was a false error, I was greatly relieved.

The other thing was the third issue in Lighthouse which identified unexpected elements were being used in the list elements. I didn’t bother to fix this as it was a low priority and I was concerned that I would break functionality by changing elements in the list.

Use of Bootstrap

I think using Bootstrap is helpful in providing a template to create the website. For the most part, the template supported accessibility. The problems I had were related to not filling in labels and alt text which was based on my customization of the template.
